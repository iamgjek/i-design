## Problem（你在找的，可能不是「只會寫 PRD」的 PM）

當團隊遇到以下狀況，多半需要的不只是文件，而是能帶動決策與交付的人：

- 開發時程常被需求牽著走，直到後期才發現風險
- UI/UX 與工程方向缺乏一致的驗證節奏，導致反覆返工
- 跨部門溝通依賴「誰說了算」，結果品質與交付難以穩定
- 面對安全、合規、或資料整合問題時，缺少可落地的治理 SOP

---

## Solution（我能為產品帶來的改變）

**1. 技術驅動決策（把風險提前變成規劃的一部分）**  
具工程背景，能預判開發難點、拆解可行路徑，並用流程化的方式降低營運與交付風險。

**2. 用戶體驗直覺（先驗證，再投入）**  
以 wireframe / prototype / User Journey / User Flow 為核心節奏，讓需求在落地前就能被快速檢驗。

**3. 數據落地（讓討論回到可衡量的結果）**  
善用 SQL / GA 等資料思維定位痛點與假設，讓「改什麼、為什麼改」更可驗證。

**4. 跨職能整合（PM / UI / Frontend 一起推進）**  
能與設計與前端並行協作，必要時也能補位交付，讓團隊在資源不足時依然維持節奏。

**5. AI 應用實踐（把 AI 變成真實產出，而非概念）**  
包含語音通話轉文字、AI 驗證與自動生成報告、知識庫生成等，提升效率並強化產品價值。
